# Troubleshooting Tools
**Diagnostic and Problem-Solving AIPL Programs**

## 🎯 Quick Problem Resolution

### **Systematic Issues**
- **File**: `systematic-debugging.yaml`
- **Use When**: Compilation errors, test failures, integration issues
- **Duration**: 15-30 minutes
- **Approach**: Layer-by-layer isolation and diagnosis

### **Missing Implementation Components**
- **File**: `incomplete-detection.yaml`  
- **Use When**: Partial implementations, missing files, incomplete service registration
- **Duration**: 5-15 minutes
- **Approach**: Systematic detection and template generation

### **XSD Schema Issues**
- **File**: `xsd-resolution.yaml`
- **Use When**: Schema validation errors, XSD integration problems
- **Duration**: 10-20 minutes
- **Approach**: Schema validation and resolution troubleshooting

## 🔧 Validation Programs

**Directory**: `validation_programs/`

### Available Programs:
- `architectural-compliance-validation.yaml` - Verify architectural patterns
- `implementation-completeness-validation.yaml` - Check implementation completeness
- `service-registration-validation.yaml` - Validate service discovery
- `snapshot-diff-integration-testing.yaml` - Integration test validation
- `snapshot-diff-registration-validation.yaml` - Registration verification

## 🚨 When to Use Each Tool

### **Start Here**: `systematic-debugging.yaml`
**If you have**:
- ❌ Compilation errors
- ❌ Test failures  
- ❌ Runtime exceptions
- ❌ Integration issues

### **Use**: `incomplete-detection.yaml`
**If you have**:
- ❓ Partial implementation
- ❓ Missing files or components
- ❓ Service registration issues
- ❓ Incomplete functionality

### **Use**: `xsd-resolution.yaml`
**If you have**:
- 📄 XSD validation errors
- 📄 Schema integration problems
- 📄 Configuration issues

### **Use Validation Programs**:
**If you need**:
- ✅ Comprehensive validation
- ✅ Architecture compliance checking

- ✅ Service registration verification

## 🔄 Problem-Solving Workflow

1. **Identify Symptoms**: What's not working?
2. **Choose Tool**: Select appropriate troubleshooting program
3. **Run Diagnosis**: Execute AIPL program with autonomous operation
4. **Apply Fixes**: Follow program's fix recommendations
5. **Validate**: Use validation programs to confirm resolution

## 🎯 Integration with Main Workflow

### **From Scenario Programs**:
If a scenario program fails, troubleshooting tools can help:
- **new-object-implementation.yaml** fails → Use `systematic-debugging.yaml`
- **enhance-existing-object.yaml** fails → Use `incomplete-detection.yaml` 
- **complete-incomplete-implementation.yaml** fails → Use validation programs

### **Autonomous Operation**:
All troubleshooting tools support autonomous operation with:
- Pre-approved diagnostic commands
- Automated fix suggestions
- Quality gates for validation

---

**Remember**: These tools are designed for systematic problem-solving, not random debugging. Start with the symptoms and choose the appropriate diagnostic approach.
