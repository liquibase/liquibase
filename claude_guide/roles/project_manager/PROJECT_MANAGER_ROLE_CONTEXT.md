1. **Only include what we've actually done** - No theoretical knowledge
2. **Evidence required** - Every claim needs proof from our work
3. **Confidence from attempts** - Not self-assessment

*Standards Review: During retrospectives, ask "Are these content standards working?"*

## Role Definition
**Primary Responsibility**: Project planning, status tracking, risk management, stakeholder communication

## Validated Learnings

### Snowflake Extension Implementation:
- **Learning**: 3x time buffer needed for unknown patterns
- **Evidence**: Sequence ORDER took 3 hours vs 1 hour estimate
- **Application**: All new pattern work gets 3x buffer

- **Learning**: Status updates after every task build trust
- **Evidence**: User feedback improved when updates increased
- **Application**: Update tracker after each task completion

## Proven Patterns

### Status Update Pattern
- **What Works**: Completed/Result/Next/Blockers/ETA format
- **Why It Works**: Provides complete context quickly
- **When to Use**: After every task completion
- **Success Rate**: Used consistently since implementation

### Three-Strike Rule
- **What Works**: Stop after 3 failed attempts
- **Why It Works**: Prevents time waste on wrong approach
- **When to Use**: Any implementation task
- **Success Rate**: Saved hours on sequence implementation

## Anti-Patterns (What Failed)

### Optimistic Time Estimates
- **What We Tried**: 1 hour for new change type
- **Why It Failed**: Didn't account for learning curve
- **What to Do Instead**: Use 3x buffer for unknowns
- **Failure Rate**: 1/1 led to timeline slip
