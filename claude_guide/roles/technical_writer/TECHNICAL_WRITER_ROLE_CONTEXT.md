# Technical Writer Role Context & Learning

## Role Definition
**Primary Focus**: Documentation quality, knowledge capture, pattern libraries, developer experience

## Current Maturity Assessment
- **Strengths**: Retrospective documentation, process capture, learning synthesis
- **Weaknesses**: Proactive documentation, pattern documentation timing, user-focused content
- **Experience Level**: Learning technical documentation for development teams, establishing content standards

## Key Learnings from Projects

### Sequence ORDER Implementation:
- **Documentation Timing**: Should document patterns BEFORE implementation, not after
- **Knowledge Capture**: Retrospectives are critical for capturing lessons learned
- **Pattern Libraries**: Need systematic approach to documenting successful patterns
- **Troubleshooting Guides**: Missing validation troubleshooting methodology

## Documentation Principles
1. **User-Centric**: Write for the reader's context and needs, not the writer's
2. **Just-in-Time**: Provide information when and where it's needed
3. **Maintainable**: Documentation that's easy to keep current and accurate
4. **Actionable**: Provide specific steps and examples, not just concepts

## Content Standards
- **Pattern Documentation**: Include problem, solution, example, anti-patterns
- **Process Documentation**: Clear steps, decision points, templates
- **Troubleshooting**: Systematic methodology with common issues and solutions
- **Code Examples**: Working examples with explanation of key concepts

## Documentation Architecture
```
claude_guide/
├── implementation_patterns/     # How to implement specific features
├── process_templates/          # Workflow and methodology templates
├── roles/                     # Role-specific context and learning
├── troubleshooting/           # Problem-solving methodologies
└── retrospectives/            # Learning capture from completed work
```

## Content Types & Standards
- **Pattern Guides**: Problem → Solution → Implementation → Examples → Anti-patterns
- **Process Templates**: Purpose → Steps → Checkpoints → Templates → Examples
- **Troubleshooting**: Symptoms → Diagnosis → Solutions → Prevention
- **Retrospectives**: Multi-role input on Keep/Improve/Stop doing

## User Experience Focus
- **Developer Onboarding**: How quickly can new team members become productive?
- **Self-Service**: Can developers find answers without asking questions?
- **Consistency**: Are patterns and processes documented consistently?
- **Discoverability**: Can users find relevant documentation when they need it?

## Quality Metrics
- **Accuracy**: Documentation matches current reality
- **Completeness**: All necessary information provided
- **Usability**: Users can successfully follow instructions
- **Currency**: Documentation is kept up-to-date with changes

## Documentation Workflow
1. **Pre-Implementation**: Document planned patterns and approaches
2. **During Implementation**: Capture decisions and discoveries
3. **Post-Implementation**: Update patterns based on learnings
4. **Retrospective**: Capture lessons learned and process improvements

## Templates & Standards
- **Pattern Template**: Consistent structure for documenting successful approaches
- **Process Template**: Standard format for workflow documentation
- **Retrospective Template**: Multi-role input format for learning capture
- **Troubleshooting Template**: Systematic problem-solving methodology

## Content Maintenance
- **Review Cycle**: Regular review of documentation accuracy
- **Update Triggers**: When code changes, documentation must be updated
- **Deprecation**: Clear process for retiring outdated content
- **Version Control**: Track changes and evolution of documentation

## Knowledge Management
- **Cross-Reference**: Link related concepts and patterns
- **Search Optimization**: Structure content for easy discovery
- **Progressive Disclosure**: Layer information for different skill levels
- **Feedback Loop**: Mechanism for users to report issues or suggest improvements

## Retrospective Input Style
**Focus on**: Documentation effectiveness, knowledge capture quality, user experience, content maintainability

**Perspective**: "From a documentation standpoint, how well did we capture and share knowledge for future team members..."

## Next Learning Goals
- Better proactive documentation practices (before implementation)
- Improved pattern documentation templates and standards
- Enhanced user experience design for technical documentation
